**Lógica Interna do Gateway:**

1.  **Receber Request:** Validar o schema básico do request (Pydantic é ótimo aqui).
2.  **Autenticação/Autorização:** Verificar user_id e suas permissões (usando o PeopleService).
3.  **Determinar Rota:**
    *   Se request_type for structured e o intent for claro, pode mapear diretamente para um serviço (bypassando o LLM para eficiência, se desejado).
    *   Se request_type for natural_language (ou structured sem intent claro):
        *   **Preparar Prompt para LLM:** Construir um prompt que inclua:
            *   A instrução principal (ex: "Você é um assistente de roteamento. Analise a solicitação do usuário e identifique a intenção e os parâmetros necessários para chamar uma das seguintes funções: ...").
            *   A lista de funções disponíveis (com descrição e parâmetros esperados - formato compatível com Function Calling do OpenAI/Anthropic/Gemini é ideal). Ex: get_delivery_status(order_id: str), create_sale(customer_id: str, items: list[dict]), list_products(category: str | None = None).
            *   O texto do usuário (payload.text).
            *   Contexto relevante (session_id, user_id, context).
        *   **Chamar LLM:** Enviar o prompt para a API do LLM escolhido.
        *   **Processar Resposta do LLM:** Extrair a função identificada e os parâmetros. Lidar com casos onde o LLM não entende ou pede esclarecimentos.
4.  **Validação de Parâmetros:** Usar Pydantic models para validar os parâmetros extraídos pelo LLM ou fornecidos diretamente.
5.  **Mapeamento e Invocação do Serviço:**
    *   Ter um registro (ex: um dicionário Python) que mapeia nomes de funções (ex: "get_delivery_status") para os métodos reais dos serviços (ex: DeliveryService.get_status).
    *   Invocar o método do serviço correspondente com os parâmetros validados. Ex: result = await delivery_service.get_status(order_id=params['order_id'], user_id=request.user_id)
6.  **Processar Resultado do Serviço:** Lidar com sucessos e exceções retornadas pelos serviços.
7.  **Formatar Resposta:**
    *   Se a solicitação original era natural_language, pode usar o LLM novamente para transformar o resultado estruturado do serviço em uma resposta textual amigável.
    *   Se a solicitação era structured (ou a UI prefere dados), formatar o resultado como JSON estruturado.
    *   Em caso de erro, formatar uma resposta de erro clara.
8.  **Retornar Resposta:** Enviar o Response Body formatado de volta para o frontend.

**Escolha do LLM:** Recomendo começar com um modelo que tenha um bom suporte a "Function Calling" ou "Tool Use" (como GPT-4/GPT-3.5-Turbo da OpenAI, Claude 3 da Anthropic, ou Gemini da Google). Isso simplifica enormemente a extração estruturada da intenção e parâmetros.

## 4. Contrato e Interações

*   **Frontend -> LLM Gateway:** Definido pela API POST /process_request (Request/Response Bodies acima).
*   **LLM Gateway -> LLM:** Prompt estruturado com descrição de funções, contexto e input do usuário. Resposta esperada é um JSON indicando a função a ser chamada e seus argumentos (ou um pedido de esclarecimento).
*   **LLM Gateway -> Serviços Backend:** Chamadas de funções/métodos Python assíncronos. Ex: await sales_service.create_order(order_data: OrderCreateSchema, current_user: User). Os serviços usam os schemas Pydantic para seus inputs/outputs internos.
*   **Serviços Backend -> MongoDB:** Interação via Motor, usando dicionários ou instâncias dos schemas Pydantic (com bibliotecas como Beanie ou adaptação manual) para ler/escrever nas coleções definidas acima.

# ./app/core/database.py

import motor.motor_asyncio
from .config import settings
from typing import Optional

class Database:
    client: Optional[motor.motor_asyncio.AsyncIOMotorClient] = None
    db: Optional[motor.motor_asyncio.AsyncIOMotorDatabase] = None

db_context = Database()

async def connect_to_mongo():
    """Conecta ao MongoDB na inicialização da aplicação."""
    print(f"Conectando ao MongoDB em {settings.MONGO_DETAILS}...")
    db_context.client = motor.motor_asyncio.AsyncIOMotorClient(settings.MONGO_DETAILS)
    db_context.db = db_context.client[settings.MONGO_DB_NAME]
    # Opcional: Testar a conexão
    try:
        await db_context.client.admin.command('ping')
        print("Conexão com MongoDB estabelecida com sucesso.")
    except Exception as e:
        print(f"Não foi possível conectar ao MongoDB: {e}")
        db_context.client = None
        db_context.db = None

async def close_mongo_connection():
    """Fecha a conexão com o MongoDB no desligamento da aplicação."""
    if db_context.client:
        print("Fechando conexão com MongoDB...")
        db_context.client.close()
        print("Conexão com MongoDB fechada.")

def get_database() -> motor.motor_asyncio.AsyncIOMotorDatabase:
    """Retorna a instância do banco de dados MongoDB."""
    if db_context.db is None:
        # Isso não deve acontecer se a conexão for feita no startup,
        # mas é um fallback ou pode indicar um erro de inicialização.
        raise RuntimeError("Conexão com o banco de dados não estabelecida.")
    return db_context.db
