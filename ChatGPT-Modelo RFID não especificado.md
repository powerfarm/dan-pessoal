*   Esses cartões podem exigir autenticação antes de permitir qualquer leitura, o que faz com que o app NFC Tools mostre como "invisível".

### **2\. Cartão de baixa frequência (125 kHz):**

*   Se o **Smart Card** for de **125 kHz** (como EM4100, T5577, etc.), ele não será reconhecido pelo **NFC Tools**, pois o app só funciona com NFC e RFID de **13,56 MHz**.
*   Os cartões de baixa frequência não transmitem dados legíveis via smartphones com NFC, que só trabalham com alta frequência.

### **3\. Cartão com UID randomizado:**

*   Algumas tags possuem um **UID randomizado** a cada leitura para dificultar clonagem, o que impede que o app identifique um UID fixo.
*   Isso é comum em cartões de segurança avançada, como alguns modelos de **MIFARE DESFire**.

### **4\. Cartão criptografado com firmware proprietário:**

*   Alguns cartões "smart" utilizam firmware personalizado, sendo reconhecidos apenas pelos leitores específicos programados para eles. Esses cartões não seguem o padrão de NFC Tools e podem ser bloqueados contra leitura não autorizada.

### **Por que isso acontece com a fechadura:**

*   A fechadura pode estar configurada para aceitar apenas **cartões proprietários** com protocolos exclusivos.
*   Pode ser que ela esteja configurada para aceitar apenas UIDs estáticos de Smart Cards.
*   Algumas fechaduras mais avançadas aceitam apenas cartões com sistemas de autenticação como chave AES ou algoritmos próprios.
