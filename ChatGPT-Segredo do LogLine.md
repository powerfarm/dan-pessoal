5) LogLine: gramática de orquestração (imperativa vs descritiva)
================================================================

*   **Descritiva (recomendada)**: você descreve **o que** quer (ex.: “treinar 3 fases, varrer LR, critério de parada X”), e o _scheduler_ concretiza. É mais segura, reprodutível e otimiza decisão.

### Proposta de gramática híbrida (EBNF simplificada)

```
Experiment    := Pipeline { Phase }
Pipeline      := "pipeline" "{" (objective | budget | evalset)* "}"
Phase         := "phase" Name  "{" (train | finetune | sweep | stop_rule)+ "}"
objective     := "objective" ":" Metric "(" Target ")"
budget        := "budget" ":" (tokens | flops | walltime)
train         := "train" ":" Model "on" Dataset "with" HParams
finetune      := "finetune" ":" BaseModel "on" Task "with" LoRA
sweep         := "sweep" ":" Param "in" Set "metric" Metric "mode" ("min"|"max")
stop_rule     := "stop" ":" "dval/dtokens" "<" Epsilon "over" Steps
```

```

  objective: val_ppl(<=2.8)
```

Loops de Controle (os “pilotos automáticos”)
============================================

1.  **Loop Interno (Step Loop)** – L2
    *   Sinais: `tokens/s`, `oom`, `nan`, `temp`.
    *   Ação: ajusta batch/grad-accum; pausa se risco de OOM/thermal.
2.  **Loop Médio (Phase Loop)** – L3
    *   Sinais: `dval/dtokens`, _train–val gap_.
    *   Ação: aplica _early-stop_, troca LR, muda currículo/seq\_len.
3.  **Loop Externo (Experiment Loop)** – L3/L4
    *   Sinais: comparação entre trials; custo/qualidade.
    *   Ação: escolhe melhor trial, arquiva artefatos, promove a “baseline”.
4.  **Loop de Negócio (Release Loop)** – L4
    *   Sinais: métricas de tarefa (hotelaria/vendas), riscos, custo.
    *   Ação: aprova release, ativa/desativa plugins por _feature flags_.

Fluxo de uma execução (exemplo)
===============================

1.  **L3** lê `.lll` (warmup → extend → lora), cria _plan_ e agenda trials.
2.  **L2** inicia treino; **L1** persiste spans/metrics; **L0** executa kernels.
3.  **Loop Médio** detecta `dval/dtokens < 0.2%` ⇒ _early-stop_, troca fase.
4.  **LoRA** foca r={8,16} e LR ao redor do melhor; escolhe vencedor.
5.  **L4** aplica política (quota ok?), assina snapshot, marca release candidate.
6.  Se plugin novo quebra, **L4** desativa plugin; tronco segue estável.
